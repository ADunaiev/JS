Основи Angular

Angular це JS фреймворк. Існує два види фреймворків:

  1. Angular.JS (або Angular-1)
  2. Angular-CLI

Концепція Angular полягає у компонентній моделі, схожей на ООП. Задачі розбиваються на певні групи, для яких створюються компоненти, що є локалізованими утворюваннями (мають локальні дані, змінні тощо - віджети) і при цьому взаємодіють з глобальним контекстом. А також реалізація паттерну MVVM, який дозволяє зв'язування (та спостереження) з автоматичним оновленням представлень (view) при зміні значень Моделі, яка знаходиться під спостереженням.

Angular впроваджується у HTML, та ставить під автоматичне спостереження блок, позначений атрібутом ng-app. Частіше за все, цим атриботом позначають увесь документ <html ng-app>. Усередині блоку, опрацьовуються вирази, для створення виразу використовуються подвійні фігурни дужки { {3 + 4} } = {{ 3 + 4 }}

Зв'язування даних відбувається через моделі. Модель може бути створена як у HTML, так і у коді. Доступ до даних моделі здійснюється через вираз з ії іменем: введено - {{username}}

Замовник: Адреса: Замовлення:
{{client}}

{{address}}

{{form}}

Контролери

Контролери - спосіб утворення та ініціалізації scope - локальної області (лексикографичного околу) застосунку. Дивись скрипт та коментарі у ньому. Окіл по-перше дозволяє створити змінні, функції, які доступні всім дочірнім елементам контролера. По-друге, надає програмний доступ до їх значень. По-третє, автоматично ставить на спостереження ці змінні (вважає іх моделями).

{{data2}}
data1 = {{data1}}

Директиви

В Angular директиви - розширення ідеї контролера зі спрощенним впровадженням у DOM, а також з можливістю включення до свого складу розмітки. Директиви включаються одним з наступних способів:

  • restrict: "A" - атрібут довільного тега
  • restrict: "C" - через ім'я класу
  • restrict: "M" - через коментар
  • restrict: "E" - через ім'я тегу (самий поширенний спосіб)

Компоненти

Компоненти - це еволюція директив, певно "обгортка" над ними. З найбільш поширеними значеннями за замовченням:

  • впровадження у теги
  • ізоляція околу
  • не потрібна функція при ініціалізації
